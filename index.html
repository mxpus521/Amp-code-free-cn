Amp

您的前沿编程代理。为终端和编辑器而生，由 Claude Opus 4.5 和 GPT-5 驱动。
> amp run
核心理念
model_training
多模型支持
Opus 4.5、GPT-5、快速模型 — Amp 智能选择最佳模型完成不同任务。
all_inclusive
无约束 Token
不限制 Token 使用量，提供最大能力和自主性，让模型充分发挥潜力。
history
线程管理
保存、分享和引用对话线程，如同版本控制般管理您的编程过程。
bolt
持续进化
无历史包袱，始终跟随最新模型发展，只保留经过验证的最佳功能。
智能模式
smart 默认
使用无约束的最先进模型 (Opus 4.5)，提供最大能力和自主性，适合复杂任务。
rush 快速
更快速、更经济，适合小型、明确定义的任务。使用 Ctrl+O 切换模式。
large 隐藏
隐藏的大型模式，适用于需要更强大推理能力的特殊任务场景。

AGENTS.md 配置
AGENTS.md
项目架构 — 描述代码结构、内部 API、构建和测试命令
自动包含 — 当前目录及父目录的 AGENTS.md 自动加载
子树配置 — 读取子目录文件时自动包含该目录的 AGENTS.md
@-引用 — 使用 @doc/style.md 或 @specs/*.md 引用文件
Globs 过滤 — 在 YAML frontmatter 中使用 globs 限定规则适用范围
个人配置 — ~/.config/amp/AGENTS.md 存放个人偏好
CLI 用法
$ amp
交互模式，开始对话
$ amp -x "your prompt"
执行模式，完成后退出
$ echo "input" | amp
管道输入作为首条消息
$ amp threads continue
继续上一个线程
$ $command
Shell 模式，执行并加入上下文
$ $$command
Incognito Shell，不加入上下文

高效使用技巧
check_circle
明确直接的提示
使用"执行 X"而非"能做 X 吗？"，保持简短专注。第一条提示至关重要，决定对话方向。
refresh
保持线程精简
每个线程专注单一任务。累积噪音时使用 Handoff 传递上下文到新线程，而非压缩历史。
link
线程引用
通过 URL 或输入 @@ 搜索引用其他线程，复用之前的技术和方案。
feedback
告诉代理如何验证
指明运行什么命令、打开什么 URL、查看什么日志来检查工作成果。
undo
编辑与撤销
编辑之前的消息会自动回滚该消息之后的所有更改，支持单文件撤销。
extension MCP 服务器
playwright semgrep sourcegraph linear monday 自定义...
